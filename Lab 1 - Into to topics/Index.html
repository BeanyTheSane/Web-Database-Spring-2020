    • What is ASP.NET Web Forms?

    This was the first programming model available for ASP.NET. The model utilizes the .NET framework and can be written in a language that supports the Common Language Runtime (CLR) such as visual basic and C#. This model is designed for Rapid Application Development (RAD) and is very designer driven. It allows you to use a drag-and-drop development model to quickly put together the front end views without extensive knowledge of html or javascript. This is possible because the framework will automagically generate most of the markup that will be sent to the browser. This is it's downfall however, because of the auto generated markup you have less control over some of the behaviors of your application.

    • What is ASP.NET MVC?

    This programming model splits the project into three different layers.
    The Model - Business Layer
    The View - Display Layer
    The Controller - Input Control
    This allows multiple teams to work on the same project without too much interference because the code is separated. Designers can work on the front end and developers can work on the back-end. This model also gives much more control over the behavior of the project by giving the user direct control over much of the front end development that the Web Form model automates. This model works well with larger and complex projects and supports unit testing among other features.

    • What is ASP.NET Core?

    This programming model is an updated version of the previous two which combines them into one cross platform, app building machine. Version 2.x supports both the .NET framework and the .NET core. Future version will strictly target the .NET core, which is where its cross platform capabilities come from. Although this seems like a setback, there is a lot of work being done to close the API gap between .NET framework and .NET core. This model also introduces Razor pages, which is a markup syntax designed to provide a smoother work-flow when adding code blocks to HTML.
